I have implemented the requested solution for the MNIST image digit recognition task using PyTorch. I created the python implementation and the requirements file in your workspace, along with an artifact that breaks down the code and explains the methodology.

Here's what was done:
Data Splitting: We load the full MNIST dataset (60,000 train and 10,000 test). The training split is further randomly divided into a 50,000 training set and a 10,000 validation set. These splits are consistent across all experiments.
Phase 1 (Overfitting): We introduce an excessively large multi-layer perceptron (MLP) architecture named LargeMLP containing roughly 1.8 million parameters. Because the model is so large, when we train it without regularization for 15 epochs, it memorizes the training data (exhibiting near 0 training loss and 100% training accuracy) while validation loss increases, which is a classic indicator of overfitting.
Phase 2 (Overcoming Overfitting): To fix the problem without changing the LargeMLP architecture (i.e. no dropout or batch norm layers), we use:
Weight Decay (L2 Regularization) applied via the Adam optimizer. This shrinks model weights making the model simpler.
Data Augmentation applied to the training set (Random Rotations and Translations), preventing the model from perfectly memorizing exact pixel values.
